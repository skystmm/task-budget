# 任务预算速查

## 30 秒填空模板

```markdown
@<agent> <任务一句话>

范围:
- <类型1> ≤ ___ 个
- <类型2> ≤ ___ 个
- 总产物 ≤ ___ 字

预算:
- 搜索轮次 ≤ ___ 轮
- 每轮工具 ≤ ___ 个
- 单次结果 > 100KB 截断或存文件

终止: 数量达上限 / 轮次达上限 / 用户说停

输出: 群里贴 Markdown 表格 / 100 字总结
```

## 5 个常见场景

| 场景 | 范围 | 预算 | 终止 |
|---|---|---|---|
| 人物/产品调研 | 3 核心 + 5 延伸 | ≤2 轮 | 收完即停 |
| A vs B vs C | 3 项 × 5 维度 | ≤2 轮 | 维度填完 |
| 论文/帖子汇总 | 5 论文 / 10 帖 | ≤3 轮(每平台 1 轮) | 数量达上限 |
| 监控/扫描 | 10 项 × 5 字段 | ≤1 轮 | **禁用"再扫一次"** |
| 探索性(无范围) | 🛑 反问 | — | — |

## ❌ 必爆红牌

- "帮我研究一下 X"  ← 范围不明确
- "越多越好"  ← 没终止
- "你看着办"  ← 必进死循环
- "再搜一次确认"  ← context 翻倍
- 工具结果 > 100KB 不截断  ← 单轮就爆

## ✅ 必做

- 写明数字上限
- 写明轮次上限
- 写明终止条件(三选一即停)
- 工具结果 > 100KB 写文件,agent 只拿摘要
