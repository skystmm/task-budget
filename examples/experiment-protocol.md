# 实验协议速查 (autoresearch-style, v0.4.1)

## 30 秒填空

```markdown
@<agent> <任务一句话>

【轮次衰减协议 - 强制】
目标: <文件路径 / JSON / 飞书文档>
单指标: 累计命中数 / 累计覆盖度 / <可量化>
轮次上限: ≤ N 轮(用户指定,默认 20)
单轮预算: ≤ 30s / ≤ 1 工具

【轮数定义】
- search_round = 调工具抓新内容算 1 轮(计入上限)
- finalize_round = 写入文件 + 验证(最多 1 次,不计入)

【终止条件 - 优先级递减】
1. 累计 ≥ 目标值 → 立即停
2. 连续 2 轮本轮新增 = 0 → 立即停
3. 轮次达到 N → 强制停

【每轮强制汇报】(只对 search_round)
search_round N (耗时 Xs):
  本轮新增: X
  累计: Y
  衰减率: Z%
  决策: keep / discard / end
  next: 下一步 或 终止-原因

【禁止】
- 隐瞒工具失败
- 用"我记忆里知道"伪造未抓取的内容
- 在衰减信号触发后继续搜

【产出格式】
JSON: {"status": "...", "search_rounds_run": N, "finalize_rounds_run": M, "total_items": X, "items": [...]}
或 Markdown 表格
```

## autoresearch 4 条铁律

1. **单一可改对象**:agent 只动一个文件/表/文档
2. **固定预算**:每次实验预算相同
3. **单指标**:一个标量,lower/upper better
4. **二元决策**:keep / discard,没有"还行"

## ⭐ 轮数档位速查

| 场景 | 推荐轮次 | 单轮工具 |
|---|---|---|
| 找 1 本具体书 | 5 | ≤ 1 |
| 普通调研/汇总 | **20(默认)** | ≤ 1 |
| 大型扫描(10+ 对象) | 50 | ≤ 1 |
| 研究级综述 | 100 | ≤ 1 |

## 衰减信号判断

```
search_round   本轮新增   累计   衰减率   决策
1              3          3      100%     keep
2              2          5      67%      keep(>30%)
3              1          6      33%      keep(>30%)
4              0          6      0%       discard
5              0          6      0%       ← 连续 2 轮 0,触发衰减终止
```

## 5 个场景速查

| 场景 | 推荐协议 | 终止条件 |
|---|---|---|
| 信息收集(单类) | 轮次衰减 / 5-20 轮 / 单轮 ≤ 1 工具 | 目标 / 衰减 / 轮次 |
| 调研对比(A vs B vs C) | 切分 + 每子任务 5-20 轮 | 各子任务独立终止 |
| 资料汇总(多类) | 切分 + 各 20 轮 | 各子任务独立终止 |
| 监控扫描 ⚠️ | 轮次衰减 / 1 轮 / 1 工具 | 立即收尾 |
| 探索性 🛑 | — | 反问用户 |

## ❌ 必爆红牌

- "帮我研究一下 X"  ← 范围不明确
- "越多越好"  ← 没终止
- "你看着办"  ← 必进死循环
- "再搜一次确认"  ← context 翻倍
- 工具结果 > 100KB 不截断  ← 单轮就爆
- 多 agent 并发开放式任务没切分  ← 雪崩
- 用"我记忆里知道"伪造数据  ← 衰减协议核心禁令

## ✅ 必做

- 写明数字上限(目标值)
- 写明轮次上限(用户决定,默认 20)
- 写明单指标(可量化)
- 写明 keep/discard/end 决策规则
- 写明终止条件(三选一)
- 工具结果 > 100KB 强制写文件
- 搜索轮 vs 收尾轮分开计数
