# 🧮 Task Budget — 开放式任务预算守门员

> autoresearch 范式 + 轮次衰减协议，防 agent 无限循环撑爆 V8 堆触发 OOM。

## 痛点

OpenClaw / Claude Code 等 agent 在执行「帮我收集/汇总/调研」类开放式任务时：

1. 反复调工具，结果塞进 context
2. Context 单调增长，V8 堆触顶
3. 多 agent 并发跑，内存叠加
4. **OOM killer 盲杀 worker → 系统 reboot**

2026-06-24 的巴拉巴基任务事故就是这剧本。

## 核心方案

### 轮次衰减协议（autoresearch 范式）

用**新增数变化率**作为终止器，不让 LLM 自己决定"还要不要继续"：

```
轮次   本轮新增   累计   衰减率   决策
1      3          3      100%     keep
2      2          5      67%      keep
3      1          6      33%      keep
4      0          6      0%       ← 衰减触发，立即停
```

### 三层终止条件（叠加生效）

| 优先级 | 条件 | 触发 |
|--------|------|------|
| 🥇 | 累计 ≥ 目标值 | 任务完成，立即停 |
| 🥈 | 连续 2 轮新增 = 0 | 衰减到位，立即停 |
| 🥉 | 轮次达到上限（默认 20） | 强制兜底停 |

### 轮数用户可配

| 场景 | 推荐轮次 | 说明 |
|------|---------|------|
| 找 1 本具体书 | 5 | 高确定性 |
| 普通调研/汇总 | **20（默认）** | 90% 场景覆盖 |
| 大型扫描（10+ 对象） | 50 | 中等密度 |
| 系统综述级别 | 100 | 研究级 |

### 五步工作流

```
┌─ 1. 用户发起开放式任务 ─────────────┐
│ "帮我收集 X 的资料"                   │
└────────┬──────────────────────────────┘
         ↓
┌─ 2. 守门员拦截，强制填三段 ──────────┐
│ 范围 / 预算 / 终止条件                │
└────────┬──────────────────────────────┘
         ↓
┌─ 3. 切分子任务（多类信息时） ────────┐
│ 书籍 → sub-agent                      │
│ 论文 → sub-agent     （并行，隔离）   │
│ X帖  → sub-agent                      │
└────────┬──────────────────────────────┘
         ↓
┌─ 4. 轮次衰减执行 ──────────────────┐
│ 每轮汇报：本轮新增 / 累计 / 衰减率   │
│ 决策：keep / discard / end           │
│ 工具结果 >100KB → 写文件不进 context │
└────────┬──────────────────────────────┘
         ↓
┌─ 5. 聚合输出 ───────────────────────┐
│ 主 session 只做文件读写 + 聚合       │
│ context 不增长，永远稳态              │
└──────────────────────────────────────┘
```

### 禁止项（核心禁令）

- ❌ **知识兜底** — 工具失败 = `status: "failed"`，不允许塞"我记忆里的"内容
- ❌ 隐瞒工具失败
- ❌ 衰减触发后继续搜
- ❌ 收尾超过 1 次

## 快速开始

复制以下模板，填入数字即可：

```
@agent 帮我收集 X 的资料

【轮次衰减协议 - 强制】
目标: ~/openclaw-tasks/<id>/result.json
单指标: 累计命中数(去重)
轮次上限: ≤ 20 轮
单轮预算: ≤ 30s / ≤ 1 工具

【终止条件】
1. 累计 ≥ 10 → 立即停
2. 连续 2 轮新增 = 0 → 立即停
3. 轮次达到 20 → 强制停
```

## 配套约定

- V8 堆上限 `--max-old-space-size=768`（systemd 配置）
- Swap 2GB 兜底 RSS 进程
- 任务产物放 `~/openclaw-tasks/<id>/`
